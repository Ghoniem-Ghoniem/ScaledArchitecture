As Solution Architect, <br /></p><p><br /></p><p>I want to prepare solution to generate and send <b>SMS </b>and <b>Email </b>notifications based on specific business trigger using predefined templates.<br /></p><p><br /></p><p>Building notification module can be designed in two different approaches:<br /></p><p><br /></p><p><ol><li>Each micro-service can handle it's own notification logic, which will handle the generation, sending and orchestration for all notification process. in this case, the notifications process and data will be distributed in all micro-services and it will lose the<b> Segregation of Duties (SOD)</b> concept, and it is hard to be maintainable and adding any new features. </li><br /></p><p></ol><br /></p><p><ol><li>A dedicated Notification micro-service that can handle all Notification templates generations and sending for all other micro-services, in that case we can build all notifications logic once in this micro-service. In this case we will guarantee that all features and enhancements applied once with no gaps.we can also build a dedicated front-end over this micro-service for template monitoring and modification. </li><br /></p><p></ol><br /></p><p>From above, we will go with the second approach as it's clearly will fulfill our requirement.<br /></p>
