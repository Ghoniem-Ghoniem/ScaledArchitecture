01- Data Sync Model Functional Proposed Medium Medium Moderate
Details:
As a Software provider,<br /></p><p><br /></p><p>I need to assure the parallel run of both applications E-Traffic and DX and maintain the data synchronization, so we can perform transactions from both solutions, synchronization is required due to:<br /></p><p><ul><li><b>Parallel Run</b>: both applications are up and running at the same time </li><br /></p><p><li><b>Lock User Products</b>: due to parallel running, we need to guarantee that both systems block each other if the system is interfering with the same product at the same time </li><br /></p><p></ul><br /></p>
02- Realtime update Functional Proposed Medium Medium Moderate
Details:
As a Software provider,<br /></p><p><br /></p><p>I have to assure a real-time update for running transactions, so I can have a smooth journeys/transactions with no delay.<br /></p><p><br /></p><p>We chose <b>Apache Kafka</b> as its have a plenty of capabilities that will assure the fulfillment of our this requirement, such as<br /></p><p><ul><li><b>Real-time Data Processing</b>: Kafka process data/messages in pipelines consisting of multiple stages/phases, where message input is consumed from Kafka topics and then aggregated, transformed into new topics for further consumption, or follow consumer login to persist the message. </li><br /></p><p><li><b>Data Persistency</b>: the data/messages transferred between two solutions can be persisted for 7 days <i><u>(default duration and can be changed</u></i>) on (n) replicas (default is 3 replicas).   </li><br /></p><p><li><b>Scalable</b>: we can scale the number of partitions to handle the load of messages.   </li><br /></p><p><li><b>Handle Stream Processing</b>: we can send batch of messages which will lead to push more messages in less time.   </li><br /></p><p></ul><br /></p>
03-Seamless integration, regardless the business/datamodels variance Functional Proposed Medium Medium Moderate
Details:
As a Software provider,<br /></p><p><br /></p><p>I need to guarantee data consistency between both E-Traffic and DX Applications and assure data synchronization between both applications with real-time updates although the variance between both data models, so I can run both applications together till full sunset of the legacy E-Traffic Application.<br /></p><p><br /></p><p>In that solution, we used <b>Apache Kafka</b> as (<i><u>a message broker</u></i>) to handle the communication between both applications bi-directional, so each communication channel has two topics to handle both direction of synchronization, sync topic, and callback topic <i><u>(all topics for synchronization were mentioned in KAFKA component model</u></i>), in comparison to <b>ETL</b> we chose this approached due to different data model between both applications<br /></p>
04- Retrial Model Functional Proposed Medium Medium Moderate
Details:
As a Software provider,<br /></p><p><br /></p><p>I want to provide a robust re-trial mechanism for failed sync transactions/events, we build a configurable re-trial engine with initial (5 trials).<br /></p><p><br /></p><p>We propose to re-initiate the sync state of the failed transaction to the last stable state and increase the trials by 1.<br /></p><p><br /></p><p>Then we will build a monitoring screens to alert the exceeded maximum trials with failure reason.<br /></p>
05- Logging Model Functional Proposed Medium Medium Moderate
Details:
As a Software provider,<br /></p><p><br /></p><p>I want to provide a dynamic logging mechanism for all sync states/actions, we build logging engine grouped by product to assure the max efficiency in retrieving the sync history. <br /></p><p><br /></p><p>We proposed to log all action occurred on sync record along with business key such as (Emirate Id, Chassis Number) to facilitate the inquiry for these sync events.   <br /></p>
06- Monitoring Model Functional Proposed Medium Medium Moderate
Details:
As a Software provider,<br /></p><p><br /></p><p>I want to provide front-end application for monitoring the sync transactions and visualize the trends of daily processes.<br /></p><p><br /></p><p>This model will be grouped  by products and time interval as mandatory attributes, then we can search more specific by the pre-defined business keys in each product.<br /></p>
